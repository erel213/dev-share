Dev-Share is configured through environment variables defined in a `.env` file at the project root. The setup script generates this file automatically from `.env.example`.

## Backend

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `JWT_SECRET` | — | Yes | Secret key used to sign and verify JWT authentication tokens. Auto-generated by `setup.sh`. |
| `ENCRYPTION_KEY` | — | Yes | AES-256 key (64 hex characters) used to encrypt sensitive data such as environment variable values. Auto-generated by `setup.sh`. |
| `PORT` | `8080` | No | Port the backend HTTP server listens on. |
| `DB_FILE_PATH` | `./backend/devshare.db` | No | Path to the SQLite database file. In Docker, this is set to `/data/devshare.db`. |
| `ADMIN_INIT_TOKEN` | — | No | Optional token to protect the `/admin/init` endpoint. When set, the setup wizard requires this token to create the initial admin account. |
| `TEMPLATE_STORAGE_PATH` | `./template_storage` | No | Directory where uploaded Terraform template files are stored. |
| `ENV_EXECUTION_PATH` | `./env_executions` | No | Working directory for Terraform plan and apply operations. |
| `TF_PLUGIN_CACHE_DIR` | — | No | Directory for caching Terraform provider plugins. Speeds up repeated operations by avoiding re-downloads. |
| `CORS_ALLOW_ORIGINS` | `localhost` | No | Comma-separated list of allowed CORS origins. |
| `BODY_LIMIT_BYTES` | — | No | Maximum HTTP request body size in bytes. |

## Frontend

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `VITE_API_BASE_URL` | `http://localhost:8080` | No | Base URL for API requests. Set this when running the frontend dev server locally. In Docker, this is empty because Nginx proxies API requests to the backend container. |

## Docker

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `APP_PORT` | `3000` | No | Host port mapped to the frontend container. Access the application at `http://localhost:<APP_PORT>`. |

## Cloud provider credentials

These variables are only needed when running in Docker and provisioning infrastructure with Terraform. See [Configure cloud credentials]({{< ref "docs/getting-started/cloud-credentials" >}}) for setup instructions.

### AWS

| Variable | Description |
|----------|-------------|
| `AWS_ACCESS_KEY_ID` | AWS access key ID |
| `AWS_SECRET_ACCESS_KEY` | AWS secret access key |
| `AWS_SESSION_TOKEN` | Session token for temporary credentials (STS, SSO) |
| `AWS_DEFAULT_REGION` | Default AWS region for resource provisioning |

### GCP

| Variable | Description |
|----------|-------------|
| `GOOGLE_APPLICATION_CREDENTIALS` | Path to the service account key JSON file inside the container |
| `GOOGLE_PROJECT` | GCP project ID |

### Azure

| Variable | Description |
|----------|-------------|
| `ARM_CLIENT_ID` | Azure service principal application ID |
| `ARM_CLIENT_SECRET` | Azure service principal secret |
| `ARM_TENANT_ID` | Azure Active Directory tenant ID |
| `ARM_SUBSCRIPTION_ID` | Azure subscription ID |
